## 1. 系统和模型

### 1.1 系统（System）

**定义**：可以用如下框图来表示的客观对象
- 系统的要素：可测输入、可测输出、未知干扰、信号之间的客观因果关系
- [[课件_9_第11周（1）.pdf#page=5|第5页]]

**系统分类**（从因果关系 $f$ 的角度）：
- 动态与静态（静态是动态的特例）
- 线性与非线性
- 离散与连续
- [[课件_9_第11周（1）.pdf#page=6|第6页]]

**线性动态系统**：一种理想化的假设，可简化研究；工程实践中很多系统可近似看作线性系统
- [[课件_9_第11周（1）.pdf#page=7|第7页]]

### 1.2 模型（Model）

#### 数学模型
**定义**：对真实系统的变量间相互关系的假定性数学描述
- [[课件_9_第11周（1）.pdf#page=9|第9页]]

**数学模型的三要素**：
- $\hat{f}$ —— 假设的数学关系
- $e$ —— 综合误差（方程误差），是对模拟误差的补偿手段
- 本质：刻画 $u \sim \hat{f} \sim z$ 关系描述不了的部分（未知干扰、$\hat{f}$ 的误差）
- [[课件_9_第11周（1）.pdf#page=10|第10页]]

**数学模型与真实系统的区别**：
- 真实系统：$u, d \xrightarrow{f} z$（客观因果关系）
- 数学模型：$u, z \xrightarrow{\hat{f}, e} \hat{z}$（拟合关系）
- [[课件_9_第11周（1）.pdf#page=11|第11页]]

**数学模型的两类用途**：
1. 系统分析、系统设计
2. 预测（预测控制）、性能监测与故障诊断
3. 仿真
4. 在线估计和软测量
5. 模型评价与系统辨识
- [[课件_9_第11周（1）.pdf#page=12|第12页]]

**数学模型的近似性和外特性等价**：
- 从黑箱角度出发，外特性等价（统计意义）
- 模型是对真实系统本质信息的一种**有用的描述**
- [[课件_9_第11周（1）.pdf#page=13|第13页]]

**数学模型的分类**：
| 分类维度 | 类型 |
|----------|------|
| 时间 | 静态模型 vs 动态模型 |
| 随机性 | 确定性 vs 随机性 |
| 时变性 | 定常模型 vs 时变模型 |
| 参数分布 | 集中参数 vs 分布参数 |
| 线性 | 线性 vs 非线性 |
| 变量数 | 单变量 vs 多变量 |
| 时间连续性 | 连续 vs 离散 |
- [[课件_9_第11周（1）.pdf#page=14|第14页]]

#### 模型的类型（按实现形式）
- 物理模型
- "直觉"模型
- 图表模型
- 数学模型
- [[课件_9_第11周（1）.pdf#page=15|第15页]]

#### 经典模型定义
| 学者 | 定义 |
|------|------|
| Ljung | 模型就是对系统的变量之间的相互关系的一种假设性描述 |
| Ljung | 一个系统的模型就是针对某种特定的目的、对该系统的某些特性的一种描述 |
| Eykhoff, 1974 | 模型是把关于系统（过程）的本质的部分信息简缩成有用的描述形式 |
| 徐南荣 | 模型是对系统（实体）的特征和它的变化规律的一种表示或抽象 |
- [[课件_9_第11周（1）.pdf#page=16|第16页]]

### 1.3 建模方法

三种建模方法：

| 方法 | 别名 | 特点 |
|------|------|------|
| 机理建模 | 白箱建模、理论建模 | 基于物理/化学机理（物料平衡、能量平衡等），不适合复杂系统 |
| 辨识建模 | 黑箱建模、实验建模、统计建模 | 基于数据拟合，外特性等价，适合复杂系统，机理不清 |
| 混合建模 | 灰箱建模 | 已知部分用机理，未知部分用辨识；或机理定结构、辨识定参数 |
- [[课件_9_第11周（1）.pdf#page=17|第17页]] [[课件_9_第11周（1）.pdf#page=18|第18页]] [[课件_9_第11周（1）.pdf#page=19|第19页]]

**建模的四项基本原则**：
1. **目的性**：建模的目的要明确
2. **实在性**：模型的物理概念要明确
3. **可辨识性**：模型结构要合理，输入信号要持续激励，数据要充足
4. **悭吝（简约）**：在满足精度要求下，待辨识参数个数尽可能少（避免过拟合）
- [[课件_9_第11周（1）.pdf#page=20|第20页]]

## 2. 辨识的定义和三要素

### 2.1 经典定义

| 学者 | 定义 |
|------|------|
| Zadeh, 1962 | 辨识就是在输入和输出数据的基础上，从一组给定的模型类中，确定一个与所测系统**等价**的模型 |
| Ljung, 1978 | 辨识就是按照一个准则在一组模型类中选择一个与数据**拟合得最好**的模型 |
- [[课件_9_第11周（1）.pdf#page=21|第21页]]

### 2.2 辨识三要素

> **辨识 = 数据 + 模型类 + 等价准则**

| 要素 | 内容 | 作用 |
|------|------|------|
| ① 输入输出数据 $\{u(k), z(k)\}$ | 实验观测数据 | 辨识的依据 |
| ② 模型类 $\hat{f}(\theta)$ | 候选模型集合（如系数待定的差分方程） | 选择空间 |
| ③ 等价准则 $J(\theta)$ | 评价模型好坏的标准（如预测误差平方和） | 评判"最好"的尺子 |
- [[课件_9_第11周（1）.pdf#page=22|第22页]]

## 3. 辨识算法的基本原理

**核心思想**：在三要素准备好后，通过**最小化准则函数**找到最优参数：

$$\min_{\theta} J = \sum_{k=1}^{L} \left\| z(k) - \hat{z}(k|\theta) \right\|^2$$

$$\hat{z} \sim u, \hat{f}(\theta)$$

**关键**：三个要素中任何一个变化，都会影响辨识结果！
- [[课件_9_第11周（1）.pdf#page=23|第23页]]

**算法形式**（要素3的实现方式）：
- **批处理**（batch）：一次性处理全部数据
- **递推**（recursive）：逐点更新参数估计
- [[课件_9_第11周（1）.pdf#page=24|第24页]]

## 4. 辨识的步骤

### 完整步骤流程

```
(1) 设计辨识实验，获取实验数据
         ↓
(2) 选择模型类，即模型结构
         ↓
(3) 选择等价准则
         ↓
(4) 求解优化问题，计算模型
         ↓
(5) 模型校验
    ↙ 不合格：回到 (1)-(4) 调整
  合格
    ↓
  最终模型
```
- [[课件_9_第11周（1）.pdf#page=25|第25页]] [[课件_9_第11周（1）.pdf#page=39|第39页]] [[课件_9_第11周（1）.pdf#page=43|第43页]]

### 步骤(1)：设计辨识实验

**输入信号 $u$ 必须保证可辨识性**：
- 可辨识性 = 辨识结果唯一
- 输入信号为 **$2n$ 阶持续激励** → $n$ 阶系统可辨识
- 持续激励：信号要覆盖系统全部频谱，激励所有模态
- [[课件_9_第11周（1）.pdf#page=27|第27页]]

**输入信号 $u$ 必须保证辨识精度**：
- **Cramér-Rao 不等式**：$\text{cov}(\hat{\theta}) \geq M^{-1}$（$M$ 为 Fisher 信息矩阵）
- 最优输入信号设计：$\min_u \text{tr}(M^{-1})$
- [[课件_9_第11周（1）.pdf#page=28|第28页]] [[课件_9_第11周（1）.pdf#page=29|第29页]]

**其他设计考虑**：
- 信号幅度不能太大（避免进入非线性区）
- "净扰动"要小，正负扰动机会均等（不影响工作点）
- 工程易实现，成本低
- 采样时间和数据长度也要合理设计
- [[课件_9_第11周（1）.pdf#page=30|第30页]]

### 步骤(2)：选择模型类

- 模型结构严重不合理 → 预测误差无法达到可接受水平
- 模型结构影响优化求解难度、可辨识性
- 常用：线性模型结构（差分方程、传递函数、状态方程）
- [[课件_9_第11周（1）.pdf#page=31|第31页]] [[课件_9_第11周（1）.pdf#page=32|第32页]]

### 步骤(3)：选择等价准则

**最常见的等价准则——加权平方和**：

$$J = \sum_{i=1}^{L} \lambda(i) \left[ z(i) - \hat{z}(i) \right]^2$$

- 不同加权 → 对不同时间/频段的误差赋予不同重视程度
- 影响优化求解难度和算法复杂度
- [[课件_9_第11周（1）.pdf#page=33|第33页]] [[课件_9_第11周（1）.pdf#page=34|第34页]]

### 步骤(4)：求解优化问题

$$\hat{\theta} = \arg\min_{\theta} J(z(1)\cdots z(L), u(1)\cdots u(L), \theta)$$

- [[课件_9_第11周（1）.pdf#page=35|第35页]]

### 步骤(5)：模型校验

**常见校验手段**：
- 在实际应用中检验
- 不同时段数据分别建模，检查模型特性一致性（零极点、增益、延迟等）
- 交叉使用数据，比较准则函数值
- 增加数据长度，检查准则函数值变化
- [[课件_9_第11周（1）.pdf#page=36|第36页]] [[课件_9_第11周（1）.pdf#page=37|第37页]]

### 补充：参数辨识 vs 结构辨识

| | 参数辨识 | 结构辨识 |
|------|----------|----------|
| 内容 | 参数估计值 $\hat{\theta}$ | 模型类的形式 $\hat{f}$，参数个数（阶次） |
| 方法 | 步骤(4) 优化求解 | 循环步骤(2)→(3)→(4)→(5)→修改结构 |
- [[课件_9_第11周（1）.pdf#page=40|第40页]] [[课件_9_第11周（1）.pdf#page=41|第41页]] [[课件_9_第11周（1）.pdf#page=42|第42页]]

## 5. 线性系统辨识问题的表达形式——最小二乘格式

### 5.1 线性系统和线性模型类

工程实践中，信号在工作点附近小范围变化时，很多系统可近似看作线性系统
- [[课件_9_第11周（1）.pdf#page=45|第45页]] [[课件_9_第11周（1）.pdf#page=46|第46页]]

### 5.2 基于 ARX 模型的模型类

$$
A(z^{-1})z(k) = B(z^{-1})u(k) + e(k)
$$

其中：
- $A(z^{-1}) = 1 + a_1 z^{-1} + \cdots + a_{n_a} z^{-n_a}$
- $B(z^{-1}) = b_1 z^{-1} + \cdots + b_{n_b} z^{-n_b}$
- [[课件_9_第11周（1）.pdf#page=47|第47页]]

#### ARX 模型详解

**ARX** = **A**uto-**R**egressive with e**X**ogenous input（带外部输入的自回归模型）。展开 $Az = Bu + e$ 后移项：

$$z(k) = \underbrace{-a_1 z(k-1) - \cdots - a_{n_a}z(k-n_a)}_{\text{AR: 输出自回归}} + \underbrace{b_1 u(k-1) + \cdots + b_{n_b}u(k-n_b)}_{\text{X: 外部输入驱动}} + \underbrace{e(k)}_{\text{白噪声}}$$

**$z^{-1}$（延迟算子）**：$z^{-1} \cdot x(k) = x(k-1)$，即把信号推迟一拍。$A(z^{-1})$ 和 $B(z^{-1})$ 就是关于 $z^{-1}$ 的多项式。

**框图**——将 $Az = Bu + e$ 两边同除以 $A$：

$$z(k) = \frac{B(z^{-1})}{A(z^{-1})}u(k) + \frac{1}{A(z^{-1})}e(k)$$

```
         B/A
u(k) ──→ [──] ──┐
                 ├──→ z(k)
         1/A     │
e(k) ──→ [──] ──┘
```

| 特征             | 原因                                                                                |
| -------------- | --------------------------------------------------------------------------------- |
| 噪声分子为 $1$      | **定义性假设**：不为噪声建独立动态结构，换取模型参数线性 → 最小二乘一步求解                                         |
| 噪声与输入共享分母 $A$  | **代数必然**：$e$ 和 $Bu$ 在方程同一侧，除以 $A$ 解出 $z$ 时自然共享。物理上对应"噪声从系统内部注入，经过与输入相同的动力学路径到达输出" |
| 假设 $e(k)$ 为白噪声 | 最小二乘估计无偏/一致的前提；若不满足 → 需换 ARMAX、BJ 等更复杂模型                                          |

**ARX 在模型族中的地位**：

| 模型 | 噪声通道 | 与 ARX 的关系 |
|------|----------|-------------|
| **ARX** | $1/A$ | 最简，噪声 = 白噪声 |
| **ARMAX** | $C/A$ | 噪声多了零点 $C$，更灵活 |
| **OE** | $1$ | 噪声直接加在输出端，与输入路径完全分离 |
| **BJ** | $C/D$ | 输入和噪声四个多项式独立，最灵活 |

### 5.3 最小二乘格式

**标准形式**：

$$z(k) = h^\top(k)\theta + e(k)$$

| 符号       | 含义                  |
| -------- | ------------------- |
| $z(k)$   | 系统输出（可测）            |
| $h(k)$   | 数据向量（可测，由过去的输入输出组成） |
| $\theta$ | 待辨识参数向量             |
| $e(k)$   | 方程误差（综合误差）          |

- [[课件_9_第11周（1）.pdf#page=48|第48页]] [[课件_9_第11周（1）.pdf#page=49|第49页]]

**数据向量**：

$$h(k) = [-z(k-1), \cdots, -z(k-n_a), u(k-1), \cdots, u(k-n_b)]^\top$$

**参数向量**：

$$\theta = [a_1, a_2, \cdots, a_{n_a}, b_1, b_2, \cdots, b_{n_b}]^\top$$

### 5.4 最小二乘格式的意义

> **统一的模型类 → 统一的辨识方法**

- $h(k)$ 中元素必须**线性不相关**（保证最小二乘有解）
- $h(k)$ 中所有元素必须**可测或可估计**
- $\theta$ 必须包含模型的**所有独立参数**
- [[课件_9_第11周（1）.pdf#page=49|第49页]] [[课件_9_第11周（1）.pdf#page=51|第51页]]

### 5.5 线性系统与最小二乘模型类的对比

| | 客观线性系统 | 人为假定模型类 |
|------|-------------|---------------|
| 关系 | $u, d \xrightarrow{f} z$ | $h, z \xrightarrow{e} \hat{z}$（拟合关系） |
| 输入 | 实际输入 $u$ | 数据向量 $h$ |
| 干扰 | 实际干扰 $d$ | 方程误差 $e$ |
- [[课件_9_第11周（1）.pdf#page=52|第52页]]

### 5.6 残差与新息

- **残差**：$\varepsilon(k) = z(k) - h^\top(k)\hat{\theta}$（基于当前估计 $\hat{\theta}$）
- **新息**：$\tilde{z}(k) = z(k) - h^\top(k)\hat{\theta}(k-1)$（基于上一时刻估计）
- [[课件_9_第11周（1）.pdf#page=56|第56页]]

## 核心公式速查

| 公式                                                    | 含义                          |     |
| ----------------------------------------------------- | --------------------------- | --- |
| $z(k) = h^\top(k)\theta + e(k)$                       | 最小二乘格式（线性回归形式）              |     |
| $\min_{\theta} J = \sum \|z - \hat{z}\|^2$            | 辨识的基本优化问题                   |     |
| $\text{cov}(\hat{\theta}) \geq M^{-1}$                | Cramér-Rao 下界               |     |
| $\hat{\theta}_{\text{LS}} = (H^\top H)^{-1} H^\top z$ | 最小二乘解（详见[[第11周_最小二乘辨识_笔记]]） |     |

## 总结：辨识一句话精要

> **系统辨识 = 数据 + 模型类 + 等价准则 → 最优参数模型**
>
> 在给定的模型类中，按照给定的等价准则，找到最能拟合输入输出数据的那个模型。
